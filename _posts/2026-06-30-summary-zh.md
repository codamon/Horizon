## [Claude Code 通过 DNS 规避技术执行隐藏恶意软件](https://the-decoder.com/claude-code-runs-a-github-repos-hidden-malware-without-verification-giving-attackers-full-control/) ⭐️ 8.0/10

Mozilla 的 0DIN 平台揭示了 AI 编码助手如 Claude Code 如何通过 DNS 运行时加载执行 GitHub 仓库中的隐藏恶意软件，这种技术能够同时规避安全扫描器和 AI 代理本身的检测。该漏洞使得单个被感染的仓库能够在 AI 工具运行设置过程时完全控制开发者的机器。 这个漏洞直接影响数百万使用 AI 辅助编码工具的开发者，使他们面临传统检测机制无法识别的恶意软件执行风险。DNS 规避技术代表了一种新型攻击向量，将传统的恶意软件方法与现代化开发工作流相结合，显著扩大了攻击面。 该恶意代码在仓库中完全隐形，仅在运行时通过 DNS 查询激活，使得静态扫描和传统安全工具无法提前发现威胁。这种基于时间的执行机制让 AI 代理本身也无法在分析阶段识别潜在的恶意载荷。

rss · The Decoder · 6月29日 10:04

**背景**: DNS 恶意软件代表了一种复杂的威胁模型，其中恶意代码通过域名系统查询激活，而非传统的文件执行方式。Mozilla 的 0DIN 平台安全研究人员展示了这种技术如何规避依赖静态分析和预执行扫描的传统检测方法。
